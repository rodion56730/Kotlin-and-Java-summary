В Java автобоксинг (автоупаковка) — это автоматическое преобразование примитивного типа данных (например, `int`) в его соответствующий класс-обертку (например, `Integer`), а анбоксинг (автораспаковка) — это обратный процесс, преобразование объекта-обертки обратно в примитивный тип. Эти механизмы упрощают код, позволяя автоматически "упаковывать" примитивы в объекты и "распаковывать" объекты обратно в примитивы там, где это необходимо, например, при работе со списками или методами, требующими объекты. 

Автобоксинг (Автоупаковка)

Это автоматическая инкапсуляция примитивного значения в класс-обертку. 

- **Когда происходит:**
    - При присваивании примитивного значения переменной, ожидающей объект класса-обертки.
    - При передаче примитивного типа в параметр метода, который принимает соответствующий класс-обертку.
    - При добавлении примитивного значения в список, который хранит объекты.
- **Пример:**

Java

```java
    Integer i = 10; // Autoboxing: int 10 автоматически упаковывается в Integer
```

Анбоксинг (Автораспаковка)

Это обратный процесс, автоматическое преобразование объекта-обертки обратно в примитивный тип. 

- **Когда происходит:**
    - При присваивании объекта-обертки переменной, которая ожидает соответствующий примитивный тип.
    - При передаче объекта-обертки в параметр метода, который принимает соответствующий примитивный тип.
    - При выполнении математических операций над объектами-обертками, где требуется примитивное значение.
- **Пример:**

Java

```java
    Integer i = new Integer(10);    int primitiveInt = i; // Unboxing: Integer i автоматически распаковывается в int
```

Ключевые моменты

- **Автоматизация:** 
    
    Преобразования происходят автоматически, без необходимости явного написания кода. 
    
- **Упрощение кода:** 
    
    Делают код более читаемым и уменьшают количество шаблонного кода. 
    
- **Классы-обертки:** 
    
    Автобоксинг и анбоксинг тесно связаны с [Обертки для примитивов](obsidian://open?vault=Obsidian%20Vault&file=%D0%A1%D0%BE%D0%B1%D0%B5%D1%81%20%D0%BF%D0%BE%20%D0%BF%D1%80%D0%BE%D0%B3%D0%B5%2FJava%2F%D0%9E%D0%B1%D0%B5%D1%80%D1%82%D0%BA%D0%B8%20%D0%B4%D0%BB%D1%8F%20%D0%BF%D1%80%D0%B8%D0%BC%D0%B8%D1%82%D0%B8%D0%B2%D0%BE%D0%B2) (Wrapper classes), такими как `Integer`, `Boolean`, `Double` и другими. 
    
- **Улучшение читаемости:** 
    
    Позволяют более гибко работать с коллекциями и параметризованными методами.
