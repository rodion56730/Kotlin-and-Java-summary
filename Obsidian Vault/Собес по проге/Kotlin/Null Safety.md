**Null Safety** — это «визитная карточка» Kotlin. В Java любая объектная переменная может содержать `null`, что приводит к знаменитому `NullPointerException` (NPE). Kotlin разделяет типы на те, что могут содержать `null`, и те, что нет, перенося проверку ошибок с этапа выполнения на этап компиляции.

---

# Null Safety (Безопасность Null)

## 1. Nullable и Non-Nullable типы

По умолчанию все типы в Kotlin **не могут** принимать значение `null`. Чтобы разрешить `null`, нужно добавить знак вопроса `?` после названия типа.

```Kotlin
var name: String = "Alex"
// name = null // ОШИБКА КОМПИЛЯЦИИ

var nullableName: String? = "Max"
nullableName = null // ОК
```

---

## 2. Безопасный вызов (Safe Call `?.`)

Если вы попытаетесь вызвать метод у `nullable` переменной напрямую, код не скомпилируется. Нужно использовать оператор `?.`. Если объект равен `null`, результатом всего выражения будет `null`, и программа не "упадет".

```Kotlin
val length = nullableName?.length 
// Если name == null, то length будет null (тип Int?)
```

---

## 3. Оператор «Элвис» (Elvis Operator `?:`)

Позволяет указать значение по умолчанию, если выражение слева вернуло `null`. Своим названием обязан прическе Элвиса Пресли (вопросительный знак напоминает хохолок).

```Kotlin
val len = nullableName?.length ?: 0 
// Если name == null, вернется 0. Результат — тип Int (не null).
```

---

## 4. Оператор утверждения (Not-null Assertion `!!`)

Этот оператор принудительно говорит компилятору: «Я уверен, что здесь не null, работай с этим как с обычным типом».

- **Опасно:** Если там все же окажется `null`, вы получите классический `NullPointerException`. Используйте только тогда, когда на 100% уверены в данных.

```Kotlin
val len = nullableName!!.length // Потенциальный вылет программы!
```

---

## 5. Безопасное приведение типов (`as?`)

В Java обычное приведение типов может вызвать `ClassCastException`. В Kotlin оператор `as?` вернет `null`, если приведение невозможно.

```Kotlin
val anyValue: Any = "Hello"
val intValue: Int? = anyValue as? Int // Вернет null, а не ошибку
```

---

## 6. Работа с коллекциями

Kotlin позволяет фильтровать `null` значения из списков одной командой.

```Kotlin
val listWithNulls = listOf("A", null, "B")
val cleanList: List<String> = listWithNulls.filterNotNull() // ["A", "B"]
```

---

## 7. Пример кода для 

```Kotlin
fun main() {
    val input: String? = readLine() // Может вернуть null

    // 1. Цепочка безопасных вызовов
    val city = user?.address?.city ?: "Unknown"

    // 2. Использование с let (выполнить, если не null)
    input?.let {
        // Внутри блока 'it' гарантированно не null
        println("Длина ввода: ${it.length}")
    }

    // 3. Быстрая проверка и выход
    val message = input ?: return // Завершить функцию, если input == null
    println(message.uppercase())
}
```

### Краткая шпаргалка для конспекта:

- **`String`** — никогда не `null`.
    
- **`String?`** — может быть `null`.
    
- **`?.`** — «вызови, если не null, иначе верни null».
    
- **`?:`** — «если слева null, возьми то, что справа».
    
- **`!!`** — «я рискую, дай мне NPE, если там null».
    
- **`as?`** — безопасный каст.
