# Исключения в Kotlin: Обработка и особенности

В Kotlin работа с исключениями (`Exceptions`) концептуально похожа на Java, но с одним фундаментальным отличием: **все исключения являются непроверяемыми (Unchecked)**.

### Главное отличие: Отсутствие Checked Exceptions

В Java исключения делятся на `Checked` (которые _нужно_ объявлять в `throws` или оборачивать в `try-catch`) и `Unchecked`. В Kotlin **нет ключевого слова `throws`**, и компилятор никогда не заставляет вас обрабатывать исключения принудительно.

- **Почему так сделано:** Опыт Java показал, что пустые блоки `catch` или бездумное пробрасывание исключений только загромождают код, не повышая реальную надежность.
    

---

### Конструкция Try-Catch-Finally

Синтаксис базовой обработки практически идентичен Java:

```Kotlin
try {
    // Код, который может выбросить исключение
    val result = 10 / 0
} catch (e: ArithmeticException) {
    // Обработка конкретного типа
    println("Ошибка деления на ноль: ${e.message}")
} catch (e: Exception) {
    // Общая обработка
    println("Что-то пошло не так")
} finally {
    // Выполнится всегда (закрытие ресурсов и т.д.)
    println("Завершение операции")
}
```

---

### Try как выражение (Try as an Expression)

В Kotlin `try` — это **выражение**, которое возвращает результат. Это позволяет лаконично присваивать значение переменной в зависимости от успеха операции.

- **Пример:**
    

```Kotlin
val input = "123a"
val number: Int? = try {
    input.toInt() // Если успешно, вернется число
} catch (e: NumberFormatException) {
    null // Если ошибка, вернется null
}
```

---

### Выброс исключения (Throw) и тип Nothing

Ключевое слово `throw` также является выражением. Его можно использовать, например, как часть оператора Элвиса.

- **Тип `Nothing`**: Если функция всегда выбрасывает исключение, она возвращает специальный тип `Nothing`. Это говорит компилятору, что код после вызова этой функции никогда не будет достигнут.
    
- **Пример:**

```Kotlin
val userEmail = user.email ?: throw IllegalArgumentException("Email required")

// Функция, которая всегда "падает"
def fail(message: String): Nothing {
    throw IllegalStateException(message)
}
```

---

### Ключевые моменты

- **Лаконичность:** Нет необходимости забивать код сигнатурами `throws IOException`.
    
- **Гибкость:** Использование `try` как выражения избавляет от создания временных переменных перед блоком.
    
- **Взаимодействие с Java:** Если вы вызываете метод Kotlin из Java и хотите, чтобы Java видела его как метод с `checked exception`, используйте аннотацию `@Throws`.

 ```Kotlin
    @Throws(IOException::class)
    fun readFile() { ... }
    ```


---

### Сравнение с Java

|**Функция**|**Java**|**Kotlin**|
|---|---|---|
|**Checked Exceptions**|Обязательны для обработки|Отсутствуют|
|**Try как выражение**|Нет (только оператор)|Да (возвращает значение)|
|**Тип Nothing**|Нет аналога|Есть (для функций-тупиков)|
|**Блок Finally**|Есть|Есть|
