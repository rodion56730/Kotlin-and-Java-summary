# Коллекции в Kotlin: Типы и операции

В Kotlin иерархия коллекций разделена на две ветви: **ReadOnly** (только для чтения) и **Mutable** (изменяемые). Это помогает избегать случайных изменений данных там, где это не предполагается.

## 1. Основные типы коллекций

### List (Список)

Упорядоченная коллекция элементов с доступом по индексу.

- **ReadOnly (`listOf`)**: Нельзя добавить или удалить элементы после создания.
    
- **Mutable (`mutableListOf`)**: Можно изменять состав элементов.

```Kotlin
val readOnlyList = listOf("Java", "Kotlin", "Swift")
val mutableList = mutableListOf(1, 2, 3)
mutableList.add(4) // Ок
```

### Set (Множество)

Коллекция уникальных элементов. Дубликаты игнорируются.

- **ReadOnly (`setOf`)**
    
- **Mutable (`mutableSetOf`)**
    

### Map (Словарь/Карта)

Набор пар «ключ — значение». Ключи уникальны.

- **ReadOnly (`mapOf`)**
    
- **Mutable (`mutableMapOf`)**
    

```Kotlin
val ports = mapOf("HTTP" to 80, "HTTPS" to 443)
```

---

## 2. Иерархия интерфейсов

Все коллекции наследуются от интерфейса `Iterable`.

1. **`Collection<T>`**: Базовый интерфейс (размер, проверка на пустоту, наличие элемента).
    
2. **`MutableCollection<T>`**: Добавляет методы `add`, `remove`, `clear`.
    

---

## 3. Основные операции (Функциональный стиль)

Kotlin предоставляет мощный набор функций для работы с коллекциями без использования циклов `for`.

### Преобразование (Transformation)

- **`map`**: Создает новую коллекцию, применяя функцию к каждому элементу.
    
    `val lengths = words.map { it.length }`
    
- **`associateBy`**: Создает Map, где элементы коллекции становятся значениями, а ключи вычисляются.
    

### Фильтрация (Filtering)

- **`filter`**: Оставляет только те элементы, которые подходят под условие.
    
    `val positive = numbers.filter { it > 0 }`
    
- **`filterIsInstance<T>()`**: Оставляет только элементы определенного типа.
    

### Проверки (Predicates)

- **`any`**: Хотя бы один элемент подходит под условие (возвращает Boolean).
    
- **`all`**: Все элементы подходят под условие.
    
- **`none`**: Ни один элемент не подходит.
    

### Поиск и агрегация

- **`find`**: Возвращает первый подходящий элемент или `null`.
    
- **`first` / `last`**: Возвращают первый/последний элемент (выдают ошибку, если коллекция пуста).
    
- **`count`**: Количество элементов, подходящих под условие.
    
- **`sum` / `average`**: Математические операции для числовых коллекций.

## 4. Особенности работы

### Индексы

Вы можете безопасно получать элементы по индексу, чтобы избежать ошибок:

- `list.getOrNull(index)` — вернет `null`, если индекс вне границ.
    
- `list.getOrElse(index) { default }` — вернет значение по умолчанию.
    

### Копирование

Если у вас есть `MutableList`, и вы хотите передать его в функцию только для чтения, вы можете использовать:

- `list.toList()` — создает **новую** неизменяемую копию.

## Ключевые моменты для Obsidian

- **Безопасность**: Используйте `listOf` по умолчанию, и переходите на `mutableListOf` только если изменения действительно необходимы.
    
- **Цепочки вызовов**: Операции можно соединять: `list.filter { ... }.map { ... }.sortedBy { ... }`.
    
- **Пустые коллекции**: Для инициализации используйте `emptyList<T>()`, `emptySet<T>()`.

```Kotlin
fun main() {
    // 1. СОЗДАНИЕ КОЛЛЕКЦИЙ
    val readOnlyList = listOf("Kotlin", "Java", "Swift") // Нельзя изменить
    val mutableList = mutableListOf("Apple", "Banana")  // Можно изменить
    mutableList.add("Orange")

    val readOnlyMap = mapOf("UI" to "Compose", "Backend" to "Ktor")
    val mutableMap = mutableMapOf(1 to "One")
    mutableMap[2] = "Two"

    val numberSet = setOf(1, 2, 2, 3) // Результат: [1, 2, 3] (только уникальные)

    // 2. ФИЛЬТРАЦИЯ (Filtering)
    val numbers = listOf(-5, 10, 22, 3, 0, 8)
    val positiveNumbers = numbers.filter { it > 0 } // [10, 22, 3, 8]
    val evenNumbers = numbers.filter { it % 2 == 0 } // [10, 22, 0, 8]

    // 3. ПРЕОБРАЗОВАНИЕ (Transformation)
    val doubled = numbers.map { it * 2 } // Умножаем каждый элемент на 2
    val stringRepresent = numbers.map { "Число: $it" } // Превращаем в список строк

    // 4. ПРОВЕРКИ (Predicates)
    val hasNegative = numbers.any { it < 0 }   // true (хотя бы одно меньше 0)
    val allPositive = numbers.all { it > 0 }   // false (не все больше 0)
    val noneGreater100 = numbers.none { it > 100 } // true (ни одного больше 100)

    // 5. ПОИСК И АГРЕГАЦИЯ
    val firstGreaterTen = numbers.find { it > 10 } // 22 (первое подходящее или null)
    val sumOfAll = numbers.sum()                   // Сумма всех чисел
    val countPositive = numbers.count { it > 0 }   // 4 (количество подходящих)
    
    // Безопасное получение по индексу
    val element = numbers.getOrNull(10) ?: 0 // Если индекса 10 нет, вернет 0

    // 6. СОРТИРОВКА
    val sortedList = numbers.sorted()          // По возрастанию
    val descSorted = numbers.sortedDescending() // По убыванию

    // 7. ЦЕПОЧКИ ВЫЗОВОВ (Самое частое использование)
    val result = readOnlyList
        .filter { it.startsWith("K") } // Оставляем слова на "K"
        .map { it.uppercase() }        // В верхний регистр
        .firstOrNull()                 // Берем первое или null

    println("Результат цепочки: $result") // KOTLIN
}
```
